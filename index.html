En este codelab, exploraremos el proceso de bastionado de un sistema operativo Debian. El bastionado es una práctica esencial en la administración de sistemas que tiene como objetivo principal fortalecer la seguridad y proteger el sistema contra amenazas cibernéticas.

Debian es una distribución de Linux muy utilizada en entornos empresariales y de servidores, lo que la convierte en un objetivo atractivo para potenciales atacantes. Para mantener nuestros sistemas Debian seguros, es fundamental implementar medidas de seguridad adecuadas.

A lo largo de este codelab, aprenderemos las mejores prácticas para asegurar un sistema Debian. Esto incluye configurar cortafuegos, gestionar usuarios y privilegios, mantener el sistema actualizado con las últimas actualizaciones, y establecer políticas sólidas de seguridad. Al aplicar estas medidas, podemos garantizar la confidencialidad, integridad y disponibilidad de nuestros datos, al tiempo que minimizamos el riesgo de compromisos de seguridad.

¡Comencemos nuestro viaje hacia un sistema Debian más seguro!
  • Objetivo: Aprender a ocultar el menú de arranque en GRUB2 para evitar que usuarios no autorizados modifiquen la secuencia de arranque.
  • Instrucciones:
  1. Abre una terminal en tu sistema Debian.
  2. Ejecuta el siguiente comando para editar el archivo de configuración de GRUB2: sudo nano /etc/default/grub
  3. Busca la línea que dice GRUB_TIMEOUT=5 y cámbiala a GRUB_TIMEOUT=0. Esto hará que el menú de arranque se oculte.
  4. Guarda los cambios y cierra el editor de texto.
  5. Ejecuta el siguiente comando para aplicar los cambios: sudo update-grub
  • Objetivo: Aprender a configurar una contraseña en GRUB2 para proteger la configuración de arranque.
  • Instrucciones:
  1. Ejecuta el siguiente comando para cifrar la contraseña que se utilizará en GRUB2: sudo grub-mkpasswd-pbkdf2
  2. Ingresa y confirma la contraseña que deseas utilizar.
  3. El comando generará una cadena cifrada. Copia esta cadena.
  4. Abre una terminal y ejecuta el siguiente comando para editar el archivo de configuración personalizado de GRUB2: sudo nano /etc/grub.d/40_custom
  5. Agrega las siguientes líneas al final del archivo, reemplazando CONTRASEÑA_CIFRADA con la cadena cifrada generada en el paso 3:
set superusers="root"
password root CONTRASEÑA_CIFRADA

  2. Ejecuta el siguiente comando para aplicar los cambios: sudo update-grub
  • Objetivo: Aprender a crear una copia de seguridad de la configuración de arranque de GRUB2.
  • Instrucciones:
  1. Abre una terminal en tu sistema Debian.
  2. Ejecuta el siguiente comando para crear una copia de seguridad de la configuración de GRUB2: sudo cp /etc/default/grub /etc/default/grub.bak
  3. Ahora tienes una copia de seguridad de la configuración original en caso de que necesites restaurarla en el futuro.
